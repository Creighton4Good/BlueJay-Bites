Ideas for app:

User side:
-	Login with Creighton email (to track user data)
-	Map & list view (to see events)
-	RSVP & show up to count as attendee (don’t track random people who walk by, but figure out policy on if they should still be allowed food)
-	For dietary restrictions use SNOMED, also incorporate allergies. Maybe have personalized user account, or way for events to only show up if it is something they can eat
-	Opt-in to notifications & text, but way to easily disable notifications (to prevent spam)

Admin/organizer side:
-	Login with Creighton email, have access
-	List of all created events, can delete/add
-	When creating event: Description for location (room number, building it’s next to, etc), event name (if relevant), other relevant info (need id to get into soccer stadium, expiration time, temperature, restaurant company if applicable), also possibly make certain events only appear for certain people based on how much attendance to events, also should be cutoff to not include big events (leftover food at Christmas at Creighton not relevant, because after event ends at scheduled time Sodexo packs up food, and also policies may differ based on events, also important to see if there’s different event types, whether it’s CUInvolved or something else)
-	Enter quantity of servings (should be range, and description of food without needing to list all individual food items)
-	Automatic expiration time, but option to change this (depending on how much food there is)
-	Option to give others admin access or remove access (if you know someone else will be present at table or they have information about the event)
-	Map to select location, can zoom to be precise inside a building, and maybe pictures to help guide if relevant

Nice to have features:
-	Admin can send personalized push notification about the event that isn’t a scheduled/automatic notification (can help if there is important update, like if original estimate of 15-20 servings changes because of a food spill, or if a power outage occurs in event building, meaning leftover food opportunity doesn’t necessary need to be cancelled, but user can then be aware of new circumstances under which to pick up their food that wasn’t included in the original description)
-	User can share event with someone who doesn’t have app (app has share option which opens more options like email/text, can select who to share and it has media in message)
-	Sort list view of events in orders (most food, closest location, etc)
-	Put Streetview as map option
-	Map extension which redirects to phone’s main map app to allow user designated app for location
-	Show students considering leftover events to see current RSVP numbers. This would allow them to make calculated decisions about which event to go to depending on availability and popularity
